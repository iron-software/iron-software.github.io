    Class PdfDocument

    Represents a PDF document with full manipulation capabilities - merge, split, edit, secure, and save. This is your PDF after rendering - now you can modify it, add security, extract content, or save to disk.

    Remarks

    PdfDocument is the result of any PDF generation in IronPDF. Once you have a PdfDocument, you can manipulate it in countless ways: add pages, remove pages, merge with other PDFs, add passwords, sign digitally, redact content, extract text, and much more.

    After rendering HTML to PDF, you don't just want to save it - you want to: - Merge multiple invoices into a batch - Add passwords and permissions for security - Extract text for indexing - Add watermarks or stamps - Sign documents digitally - Redact sensitive information This class provides all post-generation capabilities. PdfDocument implements IDisposable. ALWAYS use 'using' statements or call Dispose(): using (var pdf = renderer.RenderHtmlAsPdf(html)) { pdf.SaveAs("file.pdf"); } Failing to dispose causes memory leaks and locked files. - SaveAs() - Write to disk - Merge() - Combine multiple PDFs - Password - Add security - ExtractAllText() - Get text content - AddWatermark() - Brand your PDFs - Sign() - Digital signatures - Redact() - Remove sensitive content permanently 
    using IronPdf;

    // Generate and manipulate
var pdf = ChromePdfRenderer.StaticRenderHtmlAsPdf("
    Invoice
    ");
pdf.Password = "secret123";
pdf.SaveAs("secured-invoice.pdf");
pdf.Dispose(); // Or use 'using' statement

    • Form filling and extraction
    • Annotation management
    • Page manipulation (rotate, crop, resize)
    • Metadata editing
    • OCR for scanned documents
    • PDF/A conversion
    • Compression and optimization
    PdfDocument is NOT thread-safe. Use separate instances for concurrent operations. For parallel processing, generate PDFs in parallel but manipulate sequentially.

    Verifies digital signatures in a PDF file without loading the entire document. Efficient for batch validation or pre-screening files before processing.

var files = Directory.GetFiles("contracts/", "*.pdf");

if (!PdfDocument.VerifyPdfSignaturesInFile(largePdf))

    throw new Exception("Invalid signature - processing aborted");
}
// Now safe to load and process
var pdf = PdfDocument.FromFile(largePdf);
